Xreact is a lightweight Higher Order State Component for React.

Data flow in Xreact is observable and unidirectional.

Get Started

npm install xreact --save
# or
yarn add xreact

A quick walk through of a counter app will give you a brief idea of how Xreact work.

Features

Inspired by Redux and Functional Reactive Programming, xreact allows you to model user events, actions, and data as reactive streams. Now you can map, filter, compose, and subscribe those streams into your applicationâ€™s state.

Purely Functional, Declarative, and Monadic

In imperatively written code, you describe step-by-step how to process data. With xreact, we simply define data transformations, then compose them to form our data flow. There are no variables, no intermediate state, and no side effects in your data flowâ€™s data composition!

Composable and Reusable Plan

In Redux, reducersâ€™ use of switch statements can make them difficult to compose. Unlike reducers, the function x return is simply a function which can easily compose.

const Counter = x(plan1)(x(plan2)(CounterView))
// is the same as
const plan1_x_plan2_x = compose(x(plan1), x(plan2))
const Counter = plan1_x_plan2_x(CounterView)

what really happen behind compose is actually ES6 style mixin, so there wonâ€™t be any extra layer of HoC and no any performance overhead.

Easy to test, no need for mocks

Because UI and UI behavior are loosely coupled, you can test a React component by just passing it data. Behaviors can be tested by calling actions and then verifying the state.

import {mount} from 'enzyme'
import * as rx from 'xreact/lib/xs/rx'
import {rx as xtest} from 'xreact/lib/xtests'
const mountx = compose(mount, c => React.createFactory(X)({ x: rx }, c))

let counterX = mountx(
  <Counter />
)
          
let counter = counterX.find(Counter).getNode()
let counterView = counterX.find(CounterView)
let actions = counterView.prop('actions')
it('add intent to intent$ and go through sink$', () => {
  return new xtest
    .do([
      actions.inc,
      actions.inc,
      actions.inc,
    ])
    .collect(counter)
    .then(state => expect(state.count).toBe(3))
})

see more details about testing examples at https://github.com/reactive-react/xreact/blob/master/src/__tests__/xtest.tsx
